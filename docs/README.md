# 문서 분류 안내

## 결론
`docs`는 아래 5개 카테고리로 정리합니다. 변경 이력은 규칙에 따라 `docs/CHANGE_LOG.md`를 단일 기준으로 유지합니다.

## 카테고리
1. 요구사항: `docs/requirements/`
- 제품/기능 요구사항 문서

2. 설계: `docs/design/`
- 캐릭터/애니메이션/구현 아이디어 문서

3. 가이드: `docs/guides/`
- 개발/실행/스킬 사용 절차 문서

4. 정책: `docs/policies/`
- 데이터 수집, EXP 정책, 운영 원칙 문서

5. 릴리즈: `docs/release/`
- 패키징 결과, 배포 점검 결과 문서

## 현재 문서 맵
1. 변경 이력
- `docs/CHANGE_LOG.md`

2. 요구사항
- `docs/requirements/게임 요구사항 문서.md`

3. 설계
- `docs/design/캐릭터 디자인 전략.md`
- `docs/design/코드 기반 breathing 아이디어.md`

4. 가이드
- `docs/guides/개발 환경 구성.md`
- `docs/guides/스킬 사용 가이드.md`
- `docs/guides/OverlayBreathingPreview 실행 가이드.md`

5. 정책
- `docs/policies/PC 활동 EXP 정책.md`

6. 릴리즈
- `docs/release/릴리즈 준비 결과.md`
