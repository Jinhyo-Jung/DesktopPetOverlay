# 스킬 사용 가이드

## 결론
아래 6개 스킬을 사용하면 DesktopPetOverlay 개발의 반복 작업(환경 실행, 밸런스 조정, 오버레이 검증, 저장 호환성, 정책 정리, 패키징)을 빠르게 표준화할 수 있다.

## 스킬 목록과 간단 사용법
1. `desktop-pet-dev`
- 용도: WSL 기준 개발 루틴(구현-검증-CHANGE_LOG) 수행
- 사용 예시 프롬프트: `Use $desktop-pet-dev to implement the requested feature and run the standard checks.`

2. `overlay-window-spec`
- 용도: 투명/항상 위/드래그/클릭통과 오버레이 동작 구현 및 점검
- 사용 예시 프롬프트: `Use $overlay-window-spec to add click-through toggle without breaking drag.`

3. `pet-balance-simulator`
- 용도: 스탯 감소/회복/체력 패널티/진화 페이스 수치 검증
- 사용 예시 프롬프트: `Use $pet-balance-simulator to tune hunger and health decay for a 2-hour session target.`

4. `save-migration-guard`
- 용도: 세이브 스키마 변경 시 하위 호환 마이그레이션 설계/검증
- 사용 예시 프롬프트: `Use $save-migration-guard to add schemaVersion 2 migration for EXP fields.`

5. `pc-activity-exp-policy`
- 용도: PC 활동량 기반 EXP 정책과 개인정보 최소수집 기준 정리
- 사용 예시 프롬프트: `Use $pc-activity-exp-policy to define activity metrics and privacy-safe EXP conversion.`

6. `release-packaging-windows`
- 용도: Windows 배포물(.exe 중심) 패키징과 릴리즈 체크리스트 수행
- 사용 예시 프롬프트: `Use $release-packaging-windows to package the app and summarize release artifacts.`

## 스킬 사용 예상 시나리오
1. 신규 기능 구현 시나리오
- 상황: Feed/Clean/Play 기능을 추가한다.
- 권장 순서: `desktop-pet-dev` -> `pet-balance-simulator` -> `save-migration-guard`
- 기대 결과: 기능 구현, 밸런스 검증, 저장 포맷 안정성까지 한 번에 점검.

2. 오버레이 고급 기능 시나리오
- 상황: 클릭-통과 토글과 멀티 캐릭터를 추가한다.
- 권장 순서: `overlay-window-spec` -> `desktop-pet-dev`
- 기대 결과: 입력 충돌 최소화, 드래그/클릭/통과 동작 분리 검증.

3. PC 활동 EXP 연동 시나리오
- 상황: 활동량을 EXP로 반영하고 정책을 문서화한다.
- 권장 순서: `pc-activity-exp-policy` -> `pet-balance-simulator`
- 기대 결과: 개인정보 원칙을 지키면서 성장 속도도 과하지 않게 조정.

4. 배포 준비 시나리오
- 상황: 기능 개발 완료 후 Windows 배포물을 만든다.
- 권장 순서: `release-packaging-windows` -> `desktop-pet-dev`
- 기대 결과: 빌드 산출물 검증, CHANGE_LOG 반영, 릴리즈용 결과 정리.

5. 요구사항 문서 개발 시나리오
- 결론: 요구사항 문서 초안은 정책/스펙/밸런스 관점의 3단계로 분리해 작성하는 것이 가장 빠르다.
- 상황: `요구사항 문서.md`를 신규 작성하거나 `게임 요구사항 문서.md`를 대폭 개정한다.
- 권장 순서: `pc-activity-exp-policy` -> `overlay-window-spec` -> `pet-balance-simulator` -> `desktop-pet-dev`
- 작업 방식: 정책(데이터 수집/EXP 기준) 확정 후 오버레이 동작 요구사항을 고정하고, 마지막에 수치 밸런스 목표를 명시한다.
- 기대 결과: 기능 요구사항, 정책 제약, 수치 목표가 충돌 없이 정리된 문서를 빠르게 완성할 수 있다.
